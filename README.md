# Frontend Interview Portal

Образовательный портал для подготовки к frontend-собеседованию, где теория и практика идут вместе: пользователь читает материал и сразу закрепляет его интерактивным заданием в тексте статьи.

## Статус

- Milestone 2: `Выполнено` (архитектура упражнений + 2 типа с evaluate и тестами).
- Milestone 3: `Выполнено` (остальные 4 типа, частичная оценка, DnD для MatchPairs/OrderSteps).

- Milestone 5: `Выполнено` (контент статей, inline-встраивание упражнений, UX/accessibility полировка).
- Milestone 6: `Выполнено` (финальная документация и самопроверка перед сдачей).

## Почему выбран этот стек

- `Next.js (App Router)` — удобная маршрутизация для страниц `/articles`, `/test`, `/results` и простая SSG-генерация статей.
- `TypeScript` — строгие контракты для типов упражнений, ответов и результатов проверки.
- `MDX` — контент и интерактивные компоненты находятся в одном материале, что идеально для inline-формата.
- `Локальные данные` — для MVP не нужен backend, проще отлаживать и сдавать проект.
- `CSS Modules + CSS variables` — изоляция стилей компонентов и единая дизайн-система.
- `ESLint + Prettier` — контроль качества и единый стиль кода.
- `Vitest` — быстрые unit-тесты бизнес-логики (`evaluate`, `scoring`).

## Архитектура компонентов

1. Общая модель упражнений хранится в `src/components/exercises/types.ts`.
2. `ExerciseRenderer` по `exercise.type` выбирает нужный UI-компонент.
3. Для каждого типа есть своя чистая `evaluate`-функция в `src/lib/evaluation/*`.
4. В `inline` режиме компонент работает автономно внутри статьи.
5. В `test` режиме результат передается в `TestSessionProvider` (`src/components/test/TestSessionProvider.tsx`).
6. Агрегация результатов и рекомендации по слабым темам вынесены в `src/lib/scoring/*`.
7. Контент и упражнения разделены:
   - статьи: `content/articles/*.mdx`
   - данные упражнений: `src/lib/exercises.ts`

## Реализованные типы упражнений

- `TrueFalse`
- `MultipleChoice`
- `FillTheBlank`
- `MatchPairs`
- `OrderSteps`
- `Crossword` (кастомный компонент)

Все типы работают в двух режимах: `inline` и `test`.

## Основные маршруты

- `/` — главная
- `/articles` — список статей
- `/articles/js-core`
- `/articles/react-ts`
- `/articles/browser-interview`
- `/test` — test-режим (пошаговый проход, сохранение прогресса, пропуск вопроса)
- `/results` — финальный экран с итогом, разбором и weak topics

На всех ключевых страницах есть верхняя навигация: `Главная / Статьи / Тест / Результаты`.

## Сценарий проверки (куда нажать и что должно произойти)

1. Открыть `/articles/js-core`.
2. Прочитать теорию и решить inline-упражнения прямо по ходу текста.
3. Перейти в `/test` и пройти несколько вопросов.
4. Нажать `Не знаю, пропустить вопрос` на одном из вопросов.
5. Дойти до завершения теста и открыть `/results`.
6. Проверить, что на странице результатов есть:
   - общий итог `score/maxScore` и процент,
   - детальный разбор по вопросам,
   - пометка пропущенного вопроса,
   - блок слабых тем (weak topics).

## Соответствие требованиям задания

- `3` статьи с полезным контентом и встроенным интерактивом.
- `11` inline-упражнений в статьях (требование: минимум `10`).
- `11` вопросов в test-режиме (требование: `10–12`).
- Реализовано минимум `5` типов + кастомный `Crossword`.
- Есть частичная оценка, прогресс, финальный разбор и рекомендации.

Задача
Создать образовательный портал для подготовки к собеседованию на фронтенд разработчика, который не просто рассказывает темы, а заставляет практиковаться. Каждая тема должна быть закреплена интерактивным упражнением прямо в тексте статьи. Верстка + стили, JS, React, TS, Общие вопросы

Это не просто сайт со статьями. Это система, где теория и практика переплетены — читаешь абзац, тут же проверяешь понимание, идёшь дальше.


Ключевые проблемы для решения

1. Архитектура компонентов тестирования
Тебе нужно создать набор переиспользуемых компонентов, которые можно встраивать в любой текст. Это не просто "сделать тест" — это создать систему.

Подумать:
Как компонент получает данные (вопрос, варианты, правильный ответ)?
Как отображать состояния (не отвечено / отвечено верно / отвечено неверно)?
Как давать обратную связь (объяснение почему ответ правильный/неправильный)?
Как компоненты могут работать автономно (inline в тексте) и в составе теста (последовательность с подсчётом)?


2. Разнообразие типов проверки
Разные концепции требуют разных типов проверки. "Выбери правильный ответ" — самый простой, но не всегда лучший.

Базовые компоненты:
MultipleChoice — выбор одного или нескольких правильных ответов
MatchPairs — соединить пары (термин ↔ определение)
FillTheBlank — вставить пропущенное слово
TrueFalse — верно/неверно с объяснением
OrderSteps — расставить шаги в правильном порядке

Подумать:
Как визуально различать типы компонентов?
Как обрабатывать частично правильные ответы (3 из 4 пар соединены верно)?
Как сделать drag-and-drop для MatchPairs и OrderSteps?

3. Продвинутые компоненты
Простые тесты проверяют запоминание. Продвинутые — понимание и применение.

Идеи:
CaseStudy — ответ дан не точно, нужно поправить. Как оценивать ответ такой?
SpotTheHallucination — найти не верные места в ответе. Как подсвечивать выбранный текст?
PromptBuilder — собрать последовательность из блоков (процесс выполнения, например чтонибудь с eventLoop последовательность как будут выводиться консоль логи). Как визуализировать структуру?

Подумать:
Какие ещё типы проверки имеют смысл для темы подготовки к собеседованию?
Как реализовать ещё более экспериментальные способы проверки знаний? Попробуй реализовать.
Как сделать CaseStudy без реального AI (заготовленные ответы vs эвристики оценки)?

4. Два режима работы
Компоненты должны работать в двух режимах:

Inline-режим: компонент встроен в текст статьи. Читаешь, натыкаешься на вопрос, отвечаешь, читаешь дальше. Нет общего счёта, каждый компонент автономен.

Тест-режим: последовательность компонентов с общим подсчётом баллов, прогресс-баром, финальным результатом.

Подумать:
Как один и тот же компонент может работать в обоих режимах?
Как передавать результаты в тест-режиме (callback? context? event bus?)
Как показывать прогресс в тесте?
Что происходит в конце теста (сводка? рекомендации?)

## 1. JavaScript фундамент (core language)

1. **Execution Context и Call Stack**  
    Понимание как JS выполняет код, создаёт контексты и управляет стеком вызовов.
    
2. **Event Loop и async модель**  
    Различие microtasks/macrotasks, очереди задач, поведение Promise, setTimeout.
    
3. **Closures и область видимости**  
    Lexical scope, использование замыканий, частые ошибки при работе с циклами.
    
4. **this, bind, call, apply**  
    Контекст выполнения функции и способы его явного управления.
    
5. **Прототипная модель объектов**  
    Prototype chain, наследование, разница class vs prototype.

## 2. TypeScript (почти обязательный навык)

1. **Основы типов**  
    primitive types, union, intersection, literal types.
    
2. **Generics**  
    Создание переиспользуемых типов и функций.
    
3. **Utility types**  
    Partial, Pick, Omit, Record, ReturnType.
    
4. **Типизация React компонентов**  
    Props, children, event handlers.
    
5. **Narrowing и type guards**  
    Как TypeScript уточняет типы во время выполнения.

## 3. React фундамент

1. **Virtual DOM и reconciliation**  
    Как React обновляет UI и зачем нужен diffing алгоритм.
    
2. **Hooks базовые**  
    useState, useEffect, useRef — жизненный цикл функциональных компонентов.
    
3. **Композиция компонентов**  
    children, render props, composition вместо inheritance.
    
4. **Controlled vs uncontrolled компоненты**  
    Работа с формами и состоянием input.
    
5. **React rendering pipeline**  
    Как React планирует обновления компонентов.

## 4. Браузер и Web APIs

1. **DOM и event propagation**  
    Capture vs bubbling.
    
2. **Fetch API и HTTP**  
    Методы запросов, headers, статус-коды.
    
3. **LocalStorage / SessionStorage**  
    Хранение данных в браузере.
    
4. **Cookies и безопасность**  
    HttpOnly, SameSite, Secure.
    
5. **CORS**  
    Почему браузер блокирует cross-origin запросы.

## 5. Интервью-навыки

1. **Объяснение архитектурных решений**  
    Почему выбран определённый подход.
    
2. **Разбор реальных проектов**  
    Интервьюеры часто спрашивают о прошлом опыте.
    
3. **Live coding**  
    Решение задач на JS и React.
    
4. **Code review обсуждение**  
    Способность критически оценивать код.
    
5. **Системный дизайн фронтенда**  
    Архитектура крупного SPA.
